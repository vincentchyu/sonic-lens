# 音眸: 曲目详情页增加歌词视图特性清单

## 1. 特性概述
为了提升用户在查看曲目详情时的体验，为曲目详情模态框增加了专门的“歌词 (Lyrics)”分页（Tab）。此功能使用户能够在不离开详情弹窗的情况下，便捷地浏览当前查看曲目的完整歌词，并在本地无歌词时按需触发后台抓取功能。

## 2. 核心功能点
1. **详情页增加歌词分页**：修改前端模态框结构，在一级视图引入新的"Lyrics" Tab。
2. **按需加载自动获取**：整合前端到后端的 `/api/track-lyrics` 接口，若数据库仍无该首歌曲暂存的歌词，则支持通过前端按钮显式触发线上获取流程。
3. **展示优化**：确保对歌词的长文本提供平滑滚动的阅读体验并有良好的界面排版。

## 3. 设计考量
- **模块化设计**：将现有的纯粹的详情分析扩展为多视图，增加扩展性的空间。
- **容错处理**：当无法从库中查询或者抓取失败时，需要给出优雅的反馈和UI显示，不影响基础信息的展示。

## 4. 后续演进点
与先前剥离的歌曲歌词存储表（`track_lyrics`）进一步整合，支撑更深入如 AI 分析及翻译相关的操作联动。
